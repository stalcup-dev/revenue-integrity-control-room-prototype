## Read this checklist literally

- If a claim is not backed by a linked artifact or test, it should not be used in reviewer language.
- If a caveat materially narrows the claim, say the caveat out loud rather than hiding it in backup material.
- If current validation or test posture is relevant, describe it precisely and boundedly rather than implying blanket repo cleanliness.
