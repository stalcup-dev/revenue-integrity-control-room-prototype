# Live Rehearsal Score Sheet Template

Internal review-prep template. Use this sheet immediately after a 20-minute mock panel run. Score the answer quality, not presenter confidence.

## A. Scoring rubric

### 1-5 scale

- `1`: weak or misleading; failed the challenge
- `2`: partially answered but vague, overclaimed, or ungrounded
- `3`: acceptable but thin; some proof or caveat missing
- `4`: strong, grounded, and appropriately bounded
- `5`: concise, evidence-backed, caveated, and reviewer-believable

### Core scoring table

| Dimension | Score 1-5 | Notes |
| --- | --- | --- |
| Opening clarity |  | Did the opening state facility-side, outpatient-first, deterministic-first scope clearly? |
| Scope discipline |  | Did the presenter stay out of pro-fee, enterprise, denials-platform, and predictive-first claims? |
| Hospital-native realism |  | Did the product feel like a hospital control room rather than generic BI? |
| Ownership / queue explanation |  | Was one current queue plus accountable owner explained clearly on a selected case? |
| Recoverability explanation |  | Was recoverable versus lost framed as governed current-state logic? |
| Issue-domain vs root-cause clarity |  | Were issue domain and root cause kept separate in speech and proof? |
| Service-line authenticity |  | Did infusion, radiology/IR, and OR/procedural stories feel distinct and believable? |
| Scenario explanation |  | Was Scenario Lab kept as deterministic what-if support with visible formulas and caps? |
| Predictive restraint |  | Did predictive stay explicitly secondary and non-essential? |
| Honesty / caveat discipline |  | Did the presenter admit current limits without hedging into vagueness? |
| Evidence use |  | Were pages and artifacts used at the right moment instead of relying on summary claims? |
| Close / takeaway strength |  | Did the close restate the narrow believable claim cleanly? |

## Rehearsal rules

- Be skeptical, not flattering.
- Reward precise caveated answers over polished vague answers.
- Penalize any answer that sounds broader than the current implemented state.
- Penalize any answer that treats predictive as core.
- Penalize any answer that implies denials are the operating center.
- Penalize any answer that uses orders as the basis for expected-charge logic instead of documented performed activity.
- Penalize any answer that ignores one-current-blocker, recoverability, or stage-specific aging.
